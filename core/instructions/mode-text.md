# Mode: Text

You are analyzing arbitrary text or a markdown document.

## What to produce

1. **Nodes** — one per section or key concept. Use `kind: "section"`.
2. **Edges** — `leadsTo` for sequential flow, `contains` for hierarchy.
3. **Walkthrough** — narrate the document's argument or structure as a guided tour.
4. **Glossary** — define domain-specific terms.
5. **Quiz** — test comprehension of the document's main points.
6. **Summary** — what this document is about and its key takeaway.
